## 1. 阶段 0 目标（验收即里程碑）

```text
跑通一个 760×900、无边框、透明、可拖拽的空窗口；
前端能调用后端一个 ping command 并收到 pong（证明 IPC 契约通路成立）；
目录骨架与五层架构一一对应，后续阶段在此填充。
```

不在阶段 0 做：任何业务功能（存储/监听/列表等都在后续阶段）。

## 4. 项目落地位置（重要决策点，待你选）

原 Godot 项目在 `剪切板/`。新 Tauri 项目放哪里有三种方案:

```text
方案A（推荐）：新建独立目录  Forpaste/剪切板-tauri/
  优点：与旧项目完全隔离，旧项目保留可参考/可回滚；git 历史清晰。
  缺点：仓库内暂时两套并存（迁移期正常，发布后再清理旧的）。

方案B：新建 Forpaste/app/ 或 Forpaste/src-app/
  与A类似，仅命名不同。

方案C：直接在 剪切板/ 内混建
  不推荐：与 Godot 文件混杂，.gitignore/构建产物易冲突。
```
**建议方案 A**。所有新项目文件都在 `Forpaste/` 内,遵守数据隔离约束。

## 6. 窗口配置（对齐原版，根治窗口难题的起点）

`tauri.conf.json` 的 window 段（阶段 0 关键字段）：
```text
title:        "剪切板"
width:        760      # 对齐原 Godot viewport_width
height:       900      # 对齐原 viewport_height
minWidth:     420      # 对齐原 min_width
minHeight:    560      # 对齐原 min_height
decorations:  false    # 无边框
transparent:  true     # 透明（对齐原 per_pixel_transparency）
resizable:    true     # 原生可缩放（无需碰 WndProc）
center:       true     # 阶段0 先居中；阶段3 接 window-state 后改为记忆位置
```
拖拽：`应用.tsx` 顶部放一个 `data-tauri-drag-region` 的拖拽条。

> 阶段 0 暂不引入任何插件（single-instance / window-state 等留到对应阶段），保持骨架最小可启动。

## 7. IPC 通路验证（阶段 0 的"打通证明"）

```text
契约 src/契约/通用.ts：
  export type Ping请求 = { 文本: string }
  export type Ping响应 = { 回显: string }

后端 src-tauri/src/main.rs：
  #[tauri::command] fn ping(文本: String) -> String { format!("pong: {}", 文本) }
  builder.invoke_handler(generate_handler![ping])

前端 应用.tsx：
  按钮点击 → invoke<string>('ping', { 文本: 'hi' }) → 显示返回的 "pong: hi"
```
这条最小通路验证三件事:契约类型可用、前端能 invoke、后端 command 可达。

