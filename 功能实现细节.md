# 功能实现细节（Tauri 重构）

> 本文件为《阶段性开发计划.md》的配套「功能实现视图」。
> 每个功能给出：**原版机制 → 新栈各层实现 → 关键点/坑**。
> 数据字段、行为均对齐原 Godot 版实际代码，确保重写不走样。
> 仍属计划稿，确认前不写实现代码。

---

## 0. 实现总原则（落到每个功能）

```text
分层落点：UI(React) → 协调(store) → 契约(TS类型) → 后端能力(Rust) → 平台(仅专属)
每个功能先问：它在哪一层？大多数功能止步于「前端 + 一个插件」，不下沉到平台层。
契约先行：任何跨前后端的调用/事件，先在 src/契约/ 写类型，再两侧实现。
命名沿用中文：与原项目一致，降低大模型理解成本。
```

**功能 → 实现方式总表**（一眼看清哪些是「纯前端」「插件」「要手写」）

| 功能 | 主要落点 | 实现方式 |
|---|---|---|
| 剪切板监听 | 后端能力 | 社区插件 `tauri-plugin-clipboard` 监听事件 |
| 剪切板读写 | 后端能力 | 官方 `clipboard-manager` |
| 历史/设置存储 | 后端能力 | `tauri-plugin-store` + 自写版本化迁移 |
| 历史列表/虚拟列表 | 纯前端 | React + 虚拟列表库 |
| 记录行/搜索/主题/状态消息 | 纯前端 | React 组件 |
| 卡片缩放 | 纯前端 | CSS 变量 + 状态 |
| 无边框/拖拽/缩放 | 配置+前端 | `tauri.conf.json` + CSS `data-tauri-drag-region` |
| 托盘 | 后端能力 | Tauri 内置 tray |
| 单实例 | 后端能力 | 官方 `single-instance` |
| 窗口状态/离屏 | 后端能力 | 官方 `window-state` |
| 全局快捷键 | 后端能力 | 官方 `global-shortcut` |
| 开机自启 | 后端能力 | 官方 `autostart` |
| **跟随插入符** | **平台层** | **Windows UIA / GUITHREADINFO（手写）** |
| **十键粘贴** | **平台层** | **写剪切板 + `enigo` 模拟 Ctrl+V（手写）** |

## 1. 数据层：存储与版本化迁移

### 1.1 原版数据结构（必须对齐）

历史文件 `clipboard_history.json`：
```json
{ "版本": 1, "历史记录": [
  { "ID": "1717400000_12345", "文本内容": "…",
    "创建时间": 1717400000, "最后使用时间": 1717400000 }
] }
```
应用设置 `app_preferences.json`：含 `窗口默认置顶 / 上次窗口位置{已保存,x,y} / 上次窗口尺寸 / 开机自启 / 唤醒时跟随插入符 / 主题模式(light|dark) / 卡片缩放比例 / 快捷键{ctrl,alt,shift,meta,keycode,keys}`。

> 注意：原版已有 `"版本":1` 字段——**新栈直接以它为迁移机制的起点**，而非从零设计。

### 1.2 新栈实现

```text
后端 src-tauri/src/能力/存储.rs
  - 用 tauri-plugin-store 管理两个 store：历史、设置。
  - 读：load → 取 schema_version → 跑迁移链 → 返回当前版本结构。
  - 写：防抖（前端攒变更，后端定时落盘）+ 退出前强制 flush。
  - 历史裁剪：超过 最大条数(默认500) 砍尾部。
契约 src/契约/数据.ts
  - 定义 历史条目 / 应用设置 / 存储结果 的 TS 类型（与后端 serde 结构一一对应）。
前端 src/协调/store
  - 持有内存态历史/设置；变更走 IPC，UI 不直接碰文件。
```

### 1.3 版本化迁移机制（决策 6.3 的落地）

```text
迁移链：migrations: [v1→v2, v2→v3, …]，每个是纯函数 (旧JSON)->(新JSON)。
读取流程：
  let v = data.schema_version ?? 1
  for m in 迁移链[v..当前] { data = m(data) }   // 逐级升级
  写回升级后的结构（顺带持久化新版本号）
失败兜底：
  - 解析失败 / 迁移抛错 → 把原文件复制为 *.bak.时间戳 → 用空数据启动 → 状态栏提示「历史已备份，从空白开始」。
  - 绝不静默覆盖或丢数据。
扩展点：预留「外部导入」接口（将来若要导旧 Godot 数据可挂这里），本期不实现导入。
```

**关键点/坑**
- ID 生成原版是「秒级时间戳_随机5位」，新栈改用 UUID 或保持同格式皆可，但**迁移时旧 ID 必须原样保留**。
- 时间戳原版是秒级 unix。前端展示格式化在 UI 层做，存储只存数字。

## 2. 剪切板监听

### 2.1 原版机制
Windows 原生剪切板监听（`AddClipboardFormatListener`），失败回退到 1 秒轮询；只关心**文本**变化；变化后调 `添加或置顶文本`：已存在则更新「最后使用时间」并移到第 0 位，否则插入新项到第 0 位。

### 2.2 新栈实现
```text
后端 能力/剪切板.rs
  - 注册 tauri-plugin-clipboard，订阅其「内容变化」事件。
  - 收到变化 → 读取文本 → emit 事件「剪切板已变化{文本}」给前端。
前端 协调层
  - 监听该事件 → 调 store.添加或置顶(文本) → store 通过 IPC 让后端存储入库 → 列表刷新。
去重/置顶逻辑（移植原 添加或置顶文本）：
  - 列表中存在相同文本 → 更新 最后使用时间，移到顶部（不新建）。
  - 不存在 → 新建条目插入顶部，裁剪到最大条数。
```

**关键点/坑**
- 只处理文本，忽略图片/文件（与原版一致）。
- **自我写回要防回环**：点击复制/十键粘贴会写剪切板，触发监听 → 必须用「忽略下一次自写事件」标志,避免把自己刚写的内容又记一遍。原版隐含此需求,新栈要显式处理。
- 社区插件监听稳定性 → 阶段 0 先做小验证（风险项）。

## 3. 历史列表 / 虚拟列表 / 记录行

### 3.1 原版机制
虚拟列表渲染（只渲染可见行）应对大量记录；记录行显示两行预览 + 底部时间；前 10 行在日期后显示十键提示（如 `Ctrl+1`）；支持方向键移焦、悬停同步焦点。

### 3.2 新栈实现
```text
组件/历史列表/    React + 虚拟列表库（如 @tanstack/react-virtual）
  - 输入：当前可见列表（全部 or 搜索结果）。
  - 只渲染视口内行，滚动复用 DOM。
组件/记录行/
  - props：条目 + 序号 + 是否聚焦。
  - 两行预览用 CSS line-clamp:2；时间用 Intl 格式化。
  - 序号 < 10 时渲染十键提示徽标（Ctrl+1..9 / Ctrl+0）。
交互（协调层 + 键盘 hook）
  - ↑/↓ 改焦点索引；hover 同步焦点索引；focus 行高亮。
```

**关键点/坑**
- 虚拟列表 + 键盘焦点要协同：焦点行若在视口外需 `scrollToIndex`。
- 十键提示的序号是**当前可见列表**的前十（搜索态下是搜索结果前十）——与十键粘贴口径一致（见 §13）。

## 4. 搜索

### 4.1 原版机制
顶部搜索框关键词过滤；实时更新；搜索后回到列表顶部；显示「找到 N 条记录」；无结果显示提示。

### 4.2 新栈实现
```text
组件/搜索栏/  受控 input，onChange → 派发关键词到 store（防抖 ~150ms）。
协调层      派生「可见列表」= 关键词为空 ? 全部 : 全部.filter(含关键词)。
            关键词变化 → 列表 scrollToTop。
状态消息    可见列表为空且有关键词 → 「未找到匹配记录」；否则「找到 N 条记录」。
```

**关键点/坑**
- 搜索是纯前端派生态，不落库、不改原始顺序。
- 大小写/空白处理保持与原版一致（原版为包含匹配）。

## 5. 主题（浅色/深色）

### 5.1 原版机制
`主题调色板` 定义浅/深两套色；卡片、搜索栏、设置面板、滚动条跟随；偏好存 `主题模式`。

### 5.2 新栈实现
```text
组件/主题/   定义 CSS 变量（--bg/--card/--text/--scrollbar…）两套。
            切换 = 给 <html> 加 data-theme="light|dark"，CSS 变量自动级联。
            读写 设置.主题模式 via store。
```
**关键点/坑**：用 CSS 变量级联，组件不各自写颜色，换肤零散点修改。滚动条用 `::-webkit-scrollbar` 跟随变量。

## 6. 复制写回 / 删除 / 清空

### 6.1 新栈实现
```text
复制写回   点击卡片/复制按钮 → clipboard-manager.writeText(文本)
           → 状态提示「已复制」。
           注意：原版点击卡片会「置顶」该项（更新最后使用时间并移顶）；
                十键粘贴则【不置顶】（见 §13）——两条路径要分清。
删除单条   store.删除(ID) → IPC 落库 → 列表刷新。
清空全部   先弹确认对话框 → 确认后 store.清空 → 落库。
```
**关键点/坑**：确认框用前端模态即可（不需要原生 dialog）；删除/清空都要触发防抖保存。

## 7. 无边框 / 拖拽 / 原生缩放（根治窗口难题）

### 7.1 原版机制（痛点来源）
Godot 自己抢 HWND、加 `WS_THICKFRAME`、手处理 `NCCALCSIZE/NCPAINT/NCACTIVATE`，还要处理句柄未就绪重试——整类补丁的源头。

### 7.2 新栈实现（从设计上消失）
```text
tauri.conf.json:
  "decorations": false   // 无边框
  "resizable": true      // 原生可缩放（系统画边框热区，无需碰 WndProc）
  "transparent": true    // 如需圆角/阴影
拖拽：顶部栏 HTML 加 data-tauri-drag-region，点击拖动即移动窗口（Tauri 内置）。
```
**关键点/坑**：**不再有任何 WndProc / HWND / 句柄时机代码**。这正是换栈的核心收益——原 4 份窗口修复计划在此一次性作废。

## 8. 托盘

### 8.1 新栈实现
```text
后端 能力/托盘.rs（Tauri 内置 TrayIcon）
  - 菜单项：显示/隐藏窗口、切换置顶、退出。
  - 左键点击托盘 → 切换窗口显隐。
  - 关闭按钮拦截：window close 事件 → 阻止默认 → 改为 hide（隐藏到托盘）。
  - 置顶：window.set_always_on_top(bool)，状态存 设置.窗口默认置顶。
  - 退出：菜单「退出」→ flush 存储 → app.exit()。
```
**关键点/坑**：关闭=隐藏、托盘退出=真退出，二者区分（对齐原「运行行为」）。退出前必须 flush 历史与设置。

## 10. 窗口状态持久化 / 离屏恢复

### 10.1 原版机制
存窗口位置/尺寸/置顶偏好；多屏切单屏时检测离屏并拉回可见区。

### 10.2 新栈实现
```text
后端：tauri-plugin-window-state —— 自动保存/恢复位置尺寸，【自带离屏修正】。
置顶偏好：单独存 设置.窗口默认置顶，启动时应用。
```
**关键点/坑**：原版手写的「离屏检测 + 拉回」由插件免费提供，是又一处「难题→插件」的收益。验收时仍要实测多屏切单屏。

## 11. 全局快捷键唤醒

### 11.1 原版机制
默认 `Ctrl+Shift+Space`；可自定义，必须含修饰键且非单键；设置面板支持录制/预览/恢复默认；顶部显示当前快捷键。

### 11.2 新栈实现
```text
后端 能力/快捷键.rs：tauri-plugin-global-shortcut
  - 注册 设置.快捷键 对应组合 → 触发 → 切换窗口显隐（+可选跟随插入符）。
  - 改快捷键：先注销旧的再注册新的。
契约：快捷键配置类型 = {ctrl,alt,shift,meta,keycode,keys}（沿用原字段）。
前端 设置面板：
  - 录制：捕获 keydown，组装修饰键+主键 → 校验（含修饰键 && 非单键）→ 预览。
  - 恢复默认 → Ctrl+Shift+Space。
  - 顶部胶囊显示 keys.join('+')。
```
**关键点/坑**：注册失败（被占用）要回提示并保留旧值。校验规则原样移植（必须修饰键、禁单键）。

## 12. 开机自启

```text
后端：tauri-plugin-autostart —— enable()/disable()/is_enabled()。
前端：设置面板开关 ↔ 设置.开机自启，切换即调插件。
```
**关键点/坑**：以插件实际状态为准回显，避免设置与系统注册不一致。

## 13. 十键粘贴（平台专属之一）

### 13.1 原版铁律（必须严格遵守）
```text
Ctrl+1..9 → 第1..9条；Ctrl+0 → 第10条。
【只读，绝不置顶】：走「读取第N条」只读路径，不调置顶/不更新最后使用时间/不改顺序。
序号口径：按【当前可见列表】前十（搜索态=搜索结果前十）。
不足N条 → 不粘贴 + 提示「没有第N条记录」；内容为空 → 不粘贴。
流程：写系统剪切板 → 模拟 Ctrl+V。
与显隐全局快捷键相互独立。
```

### 13.2 新栈实现
```text
后端 能力/快捷键.rs：用 global-shortcut 注册 Ctrl+1..0 → emit「十键触发{序号}」。
前端 协调层：收到序号 → 从【当前可见列表】取第N条（只读，不置顶）
            → 越界/空 → 状态提示并中止
            → clipboard.writeText(文本)
            → 调后端 invoke「模拟粘贴」。
平台 平台/windows：模拟 Ctrl+V（见 §15 粘贴模拟）。
```
**关键点/坑**
- **顺序污染是头号风险**：十键路径严禁碰置顶逻辑（原文档专门警告）。
- 自写回环：写剪切板会触发监听 → 用 §2 的「忽略自写」标志。
- 快捷键冲突（浏览器 Ctrl+数字）第一版按固定规则,后续再做开关。

## 14. 跟随插入符（平台专属之二）

### 14.1 原版机制
设置可开启；唤醒时窗口移到文本光标附近（垂直偏移 8px）；用 UIA `TextPattern2`/`GUITHREADINFO` 取插入符屏幕坐标；取不到则返回失败提示。

### 14.2 新栈实现
```text
平台契约（src-tauri/src/平台/mod.rs）：
  fn 获取插入符坐标() -> Result<(i32,i32), 不支持>
平台/windows：移植原 UIA TextPattern2 / GetGUIThreadInfo 取光标矩形 → 返回坐标。
能力/窗口.rs：唤醒时若 设置.唤醒时跟随插入符 →
  调平台层取坐标 → set_position(坐标 + 偏移(0,8)) → 再 show。
  取不到 → 正常居中显示 + 状态提示「无法获取插入符位置」。
平台/macos|linux：返回「不支持」，UI 优雅降级（开关可见但提示当前平台不可用）。
```
**关键点/坑**：这是唯二必须手写 Win32/UIA 的能力之一，单独子计划、单独验证。降级路径要保证不崩。

## 15. 卡片缩放 & 粘贴模拟（两个收尾细节）

### 15.1 卡片缩放（纯前端）
```text
Ctrl + +/-/0 在窗口激活时生效；步进 5%；范围 75%–200%（原版 clamp 0.75–2.0）。
实现：缩放比例存 设置.卡片缩放比例 → 映射到 CSS 变量 --scale
     → 字体/按钮/边距/卡片高度按比例（非整体 transform 拉伸，保持清晰）。
```

### 15.2 粘贴模拟（平台层，供十键用）
```text
平台契约：fn 模拟粘贴() -> Result<(),不支持>
windows：enigo 或原生 SendInput 按 Ctrl↓ V↓ V↑ Ctrl↑ 顺序，
        务必成对释放，避免 Ctrl 卡住（原文档明确验收项）。
macos/linux：空壳返回不支持。
```
