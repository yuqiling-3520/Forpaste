项目正在把 `剪切板/` 这个 Godot 4.6 剪切板/便利贴工具迁移重构到 **Tauri 2**。计划稿在项目根目录:`技术栈迁移重构计划.md`(总计划)、`阶段性开发计划.md`(6 阶段执行版)、`功能实现细节.md`(17 功能实现视图)、`阶段0子计划稿.md`(脚手架执行级)。沿用方法论「计划先行,用户确认前不写实现代码」。

**已确认决策(2026-06-14):**
- UI 框架:**React**(语料最丰富、出错率最低)。
- 剪切板内容变化监听:**引入社区插件 `tauri-plugin-clipboard`**(作为依赖下载植入,不重写插件本体)。
- 数据迁移:**不导入旧 Godot 版数据**;但存储层**从第一天起内建版本化迁移机制**(数据带 `schema_version`,读取时按迁移链升级,失败有备份兜底)。

**技术栈:** Tauri 2 + Rust(仅插件注册+薄胶水)+ TypeScript/React 前端。目标体积个位数 MB。架构五层:UI组件/前端协调/IPC契约/后端能力/平台移植。唯二平台专属能力:跟随插入符、十键粘贴(Windows 先做,mac/linux 空壳)。

**当前阶段:** 决策已定,下一步进入**阶段 0**——出阶段 0 子计划稿(Tauri+React 脚手架、空无边框窗口),经用户确认后才初始化项目。

遵守 [[data-isolation-constraint]]:所有记忆与数据只存于项目文件夹内。
