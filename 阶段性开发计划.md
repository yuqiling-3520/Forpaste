# 阶段性开发计划（Godot → Tauri 重构执行版）

> 本文件是《技术栈迁移重构计划.md》第 7 节的细化执行版。
> 沿用项目方法论：**计划先行，每阶段开始前另出子计划稿，用户确认前不写实现代码**。
> 技术栈与决策项已确认（2026-06-14）：React + 社区剪切板插件 + 不迁旧数据但内建版本化迁移。

---

## 0. 如何使用本计划

- 每个阶段含：**目标 / 任务清单 / 交付物 / 验收标准 / 依赖**。
- 阶段按依赖顺序推进；标注「可并行」的任务可在同阶段内并行。
- 每阶段结束做**验收 + 阶段执行记录**（沿用原项目惯例）。
- 「功能对照」列对应原 README 功能，确保**零功能遗漏**。

---

## 1. 全局贯穿项（不属于单一阶段，每阶段都要遵守）

```text
G1 IPC 契约先行：任何前后端交互，先在 契约/ 定义类型，再两侧实现。
G2 组件单一职责：每个 UI 组件/能力服务一个文件，可被单独重写。
G3 版本化数据迁移：所有持久化数据带 schema_version，读取时按迁移链升级。
G4 平台能力降级：UI 只调契约；平台缺能力时返回「不支持」，UI 优雅提示不崩溃。
G5 阶段验收记录：每阶段产出验收清单，逐项勾选后才进入下一阶段。
```

---

## 2. 阶段总览与里程碑

| 阶段 | 名称 | 里程碑（可感知的成果） |
|---|---|---|
| 0 | 脚手架与骨架 | 能启动一个空的无边框窗口 |
| 1 | 数据层与存储服务 | 历史/设置可持久化、可恢复、可版本升级 |
| 2 | 核心 UI 与剪切板主流程 | **可日常使用的最小可用版（MVP）** |
| 3 | 窗口形态与桌面壳 | 托盘常驻、单实例、离屏恢复，窗口难题清零 |
| 4 | 全局能力 | 快捷键唤醒、自启、卡片缩放 |
| 5 | 平台移植层 | 跟随插入符 + 十键粘贴（Windows） |
| 6 | 跨平台文档与发布 | 可发布产物 + 贡献指南 |

> **关键里程碑：阶段 2 末 = MVP**（能记录、搜索、复制、删除，已可替代原软件核心用途）。
> 阶段 3 末 = 体验对齐（托盘/窗口/单实例），阶段 5 末 = 功能对齐原版。

---

## 3. 各阶段详细计划

### 阶段 0：脚手架与骨架

**目标**：搭好可运行的 Tauri 2 + React 工程骨架，确立目录结构，跑通空无边框窗口。

**任务清单**
- [ ] 初始化 Tauri 2 + React + TypeScript 工程。
- [ ] 落地目录结构：`src/{组件,协调,契约,入口}`、`src-tauri/src/{能力,平台,入口}`。
- [ ] 配置 `tauri.conf.json`：无边框、透明、初始尺寸、`capabilities/` 权限骨架。
- [ ] 建立 IPC 契约层空骨架（一个 ping/pong command 验证前后端通路）。
- [ ] 配好开发脚本（dev 热重载、build）。

**交付物**：可 `dev` 运行的空项目、目录骨架、一条打通的 IPC 通路。

**验收标准**
- 能启动一个无边框（必要时透明）窗口。
- 前端能调用后端 ping command 并收到 pong（证明契约层通路成立）。

**依赖**：无（起点）。

---

### 阶段 1：数据层与存储服务

**目标**：实现历史与设置的持久化，**内建版本化迁移机制**（决策 6.3）。

**任务清单**
- [ ] 在契约层定义数据模型类型：历史条目、应用设置、存储设置。
- [ ] 后端存储服务：读写、防抖保存、退出强制落盘、最大条数裁剪（默认 500）。
- [ ] **版本化迁移**：数据带 `schema_version`；读取时检测→按迁移链逐级升级；失败则备份原文件 + 降级提示，绝不静默丢数据。
- [ ] 重复内容策略：重复项更新到更靠前位置（迁移原行为）。
- [ ] 不导入旧 Godot JSON（决策 6.3），但预留一个「外部导入」扩展点。

**交付物**：后端存储服务 + 契约数据类型 + 迁移链骨架（v1 起步）。

**验收标准**
- 历史与设置可持久化、重启可恢复。
- 写入一个「旧 schema 测试文件」能被正确升级到当前版本。
- 模拟迁移失败时，原文件被备份且有降级提示。

**功能对照**：默认 500 条、持久化、防抖落盘、重复内容前移。

**依赖**：阶段 0（契约层、工程骨架）。

---

### 阶段 2：核心 UI 与剪切板主流程（MVP）

**目标**：实现主界面与剪切板核心闭环——**这是最小可用版**。

**任务清单（UI 组件，多数可并行）**
- [ ] 历史列表（**虚拟列表**，应对大量记录滚动）。
- [ ] 记录行（两行预览 + 底部时间；复制/删除/选中交互）。
- [ ] 搜索栏（关键词实时过滤、搜索后回顶、显示「找到 N 条」、无结果提示）。
- [ ] 状态消息（操作提示 / 空列表 / 搜索无结果）。
- [ ] 主题（浅色/深色，卡片/搜索栏/滚动条跟随）。
- [ ] 键盘上下方向键移动焦点、鼠标悬停同步焦点。

**任务清单（主流程）**
- [ ] 接入社区剪切板监听插件（决策 6.2）：监听文本变化 → 入库 → 列表刷新。
- [ ] 点击卡片/复制按钮 → 写回系统剪切板。
- [ ] 删除单条 / 清空全部（清空需确认框）。
- [ ] 前端协调层：把「监听→存储→列表→复制」编排起来（组合根只编排）。

**交付物**：可日常使用的剪切板历史工具主界面。

**验收标准**
- 复制内容自动入库、可搜索、可点击复制、可删除/清空（清空有确认）。
- 大量记录滚动不卡（虚拟列表生效）。
- 主题切换即时生效、偏好保存。

**功能对照**：剪切板历史、搜索与列表、主题与显示（除缩放，缩放在阶段4）。

**依赖**：阶段 1（存储）、阶段 0（契约）。

---

### 阶段 3：窗口形态与桌面壳

**目标**：把它变成真正的「桌面常驻小工具」，**根治原项目整类窗口难题（不碰 WndProc）**。

**任务清单**
- [ ] 无边框 + 可缩放 + 顶部拖拽（Tauri 原生窗口 API）。
- [ ] 托盘菜单：显示/隐藏窗口、切换置顶、退出程序。
- [ ] 关闭按钮 → 默认隐藏到托盘（不退出）。
- [ ] 单实例唤起（官方 `single-instance`，须首个注册）：重复启动唤醒已有窗口。
- [ ] 窗口位置/尺寸/置顶偏好持久化 + 离屏恢复（官方 `window-state`）。

**交付物**：完整桌面壳——托盘常驻、单实例、窗口状态记忆。

**验收标准**（逐项对照原「窗口难题清单」，**且无需任何补丁**）
- 主窗口可正常隐藏/显示（原 Godot 痛点）。
- 无边框可原生缩放/拖拽（原需手处理 NCCALCSIZE 的痛点）。
- 多屏切单屏时离屏窗口自动拉回。
- 单实例唤醒生效。

**功能对照**：窗口与托盘整节。

**依赖**：阶段 2（有界面可承载）。

---

### 阶段 4：全局能力

**目标**：补齐全局快捷键、自启、卡片缩放。

**任务清单**
- [ ] 全局快捷键唤醒（官方 `global-shortcut`）：默认 `Ctrl+Shift+Space`。
- [ ] 设置面板：录制快捷键、预览、恢复默认；校验必须含修饰键且非单键。
- [ ] 顶部显示当前全局快捷键。
- [ ] 开机自启（官方 `autostart`）。
- [ ] 卡片缩放：`Ctrl + +/-/0`，步进 5%，范围 75%–200%，比例保存。
- [ ] 响应式缩放：字体/按钮/边距/卡片高度按比例调整（非简单拉伸）。
- [ ] 设置面板：置顶、自启、深色主题、跟随插入符开关、查看监听模式。

**交付物**：设置面板 + 全局快捷键 + 自启 + 缩放体系。

**验收标准**
- 全局快捷键唤醒/隐藏窗口；自定义并校验生效。
- 开机自启可开关并实际生效。
- 缩放快捷键在窗口激活时生效，比例持久化。

**功能对照**：快捷键、设置项、主题与显示中的缩放。

**依赖**：阶段 3（窗口显隐）、阶段 2（设置 UI 载体）。

---

### 阶段 5：平台移植层（唯一手写底层）

**目标**：实现唯二平台专属能力，并立好跨平台「契约 + 降级」骨架。

**任务清单**
- [ ] 在平台移植层定义契约：`获取插入符坐标`、`模拟粘贴`。
- [ ] **跟随插入符**（Windows）：唤醒时窗口移到文本光标附近；获取失败则返回失败提示。
- [ ] **十键粘贴**（Windows，`enigo` 或原生）：
  - [ ] `Ctrl+1..9` 粘贴当前列表第 1–9 条，`Ctrl+0` 粘第 10 条。
  - [ ] 先写剪切板再模拟 `Ctrl+V`；不改变历史相对顺序。
  - [ ] 搜索状态下对应搜索结果前十条；不足十条只粘存在的并提示。
  - [ ] 前十张卡片在日期后显示对应快捷键提示。
- [ ] macOS / Linux：空壳实现 + README + 能力降级（缺能力优雅提示）。

**交付物**：Windows 全功能；mac/Linux 可编译可运行（能力降级）。

**验收标准**
- Windows 跟随插入符、十键粘贴均按原版行为工作。
- 其他平台编译通过、启动不崩、缺失能力有提示。

**功能对照**：十键快速粘贴、跟随插入符、平台说明。

**依赖**：阶段 4（快捷键体系，十键复用其注册机制）、阶段 2（列表数据源）。

---

### 阶段 6：跨平台扩展文档与发布整理

**目标**：可发布 + 可被他人扩展。

**任务清单**
- [ ] 写《平台移植贡献指南》：如何为新平台填空实现。
- [ ] 图标、版本号、打包配置、三平台产物。
- [ ] 更新 README（新栈、新功能、运行行为、数据文件位置）。
- [ ] 体积核对（确认达到个位数 MB 目标）。

**交付物**：发布产物 + 贡献指南 + 更新后的 README。

**验收标准**
- 贡献者能照文档为新平台新增实现。
- 产物体积达到个位数 MB。

**依赖**：阶段 5（功能完整）。

---

## 4. 整体通过标准（对齐迁移计划第 9 节）

```text
□ 体积显著下降到个位数 MB。
□ 原「窗口控制类」难题无需补丁即正常工作。
□ 剪切板/搜索/快捷键/自启/单实例/托盘/离屏/缩放/主题全部可用。
□ 十键粘贴、跟随插入符（Windows）行为与原版对齐。
□ 每个重要功能是独立组件，可被大模型单独重写不破坏他人。
□ 平台专属能力隔离清晰，其他开发者可按文档为新平台填空。
□ 数据带版本号，未来结构升级可平滑迁移。
```

## 5. 风险与缓解（承接迁移计划第 8 节）

| 风险 | 缓解 | 处理阶段 |
|---|---|---|
| 社区剪切板插件监听稳定性 | 阶段 0/2 先做小验证，确认稳定再深用 | 0→2 |
| Rust 学习成本 | 插件化把手写 Rust 降到最小，模式固定 | 全程 |
| 跟随插入符/粘贴模拟（平台专属） | 单独子计划、单独验证 | 5 |
| 数据迁移逻辑出错 | schema_version + 备份兜底 + 测试用例 | 1 |
| Linux 渲染差异 | UI 简单，交未来 Linux 贡献者实测 | 6 |
| 仅 Windows 开发机 | mac/Linux 只保证可编译 + 降级 | 5/6 |
